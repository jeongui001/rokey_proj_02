# 이동 물체 파지 강건성 — 현재 z 처리(last_mid_z) 설명과 개선 제안

> 작성 2026-07-11. **정지/저속 대상은 현재 구현으로 잘 동작하므로 코드 변경은 보류**하고, 이후
> 이동 물체 강건성 보완이 필요해질 때를 위한 분석·제안만 정리한 문서다(구현 아님).

## 1. 지금 z 오조준을 무엇으로 배제하고 있나 — `last_mid_z`

### 배경: 왜 z만 특별 취급하나
`ToolTracker.update()`는 프레임마다 파지 기준점(anchor)의 출처를 `detection_anchor()`로 정한다:

| mode | 기준점 | 조건 |
|---|---|---|
| `mid` | 두 keypoint(p0=머리, p1=손잡이 끝)의 **중점 = 파지점** | 두 kpt conf ≥ 0.5 |
| `p0`/`p1` | 살아있는 한쪽 kpt 픽셀 + 학습된 3D 오프셋 | 한쪽만 conf ≥ 0.5 (근접 시 반대 끝이 화면 밖으로 잘려 conf가 0으로 떨어짐) |
| `bbox` | bbox 중심 | kpt 둘 다 저신뢰/없음 |

문제는 **mid가 아닌 모드의 raw z는 파지점의 depth가 아니라는 점**이다. p0/p1은 도구의 얇은 끝(손잡이 끝·머리 모서리)의 depth를 읽고 오프셋을 더하는 방식이라, 그 기준 픽셀 자체가 배경/벨트 경계에 가까워 raw z 노이즈가 mid보다 훨씬 크다. 실기 로그상 정지 물체인데도 mid 구간 z 변동이 ~1cm대였다가 p0 전환 직후 같은 물체에서 ~3cm대로 커졌고, 이동 물체 픽에서는 p0 구간 5초 동안 raw z가 1.6~12.3mm로 흔들리다 실제 표면 위쪽으로 벗어난 값에 정착 → **로봇이 허공을 잡는** 현상이 나왔다.

### 동작: mid에서 확정한 z를 얼려서 재사용
`tracking.py`의 4.5절(`update()` 내부):

```python
if chosen_mode == 'mid' and depth_valid:
    self.last_mid_z = z          # mid일 때만 z를 "신뢰 확정값"으로 저장
else:
    depth_valid = False          # mid가 아니면 이번 z는 못 믿는다고 표시
    if self.last_mid_z is not None:
        z = self.last_mid_z      # 마지막 mid 확정값으로 고정
```

핵심 두 가지:
1. **컨베이어 평면 가정** — 물체는 벨트 위 평면에 있어 접근 중 z(높이)는 사실상 불변. 그러니 한 번 mid로 제대로 잰 z를 계속 믿는 게, 못 믿을 기준점에서 매 프레임 재측정하는 것보다 낫다.
2. **`depth_valid=False` 전파** — 이 플래그가 `robot_control`의 정밀 칼만필터(`servo_loop.py::on_tool_track`)의 `update_xy_only` 경로를 타게 해, 로봇 쪽에서도 z를 갱신하지 않고 xy만 추종한다. 즉 vision과 robot_control 양쪽에서 일관되게 z를 얼린다.

### 기존 안전장치와의 분리
depth 패치 자체가 무효일 때 쓰는 `last_valid_z`(전체 계획.md 2.7절)와 **별개 변수**다. 안 그러면 "raw는 유효(depth_valid=True)하지만 파지점이 아니라 못 믿는" p0의 z가 `last_valid_z`를 오염시켜, 원래의 depth-무효 프리즈 안전장치까지 깨진다.

### 효과와 한계
- **효과**: 정지/저속 대상에서 mid→단일kpt 전환 시의 z 급변(허공 파지)을 효과적으로 배제. 실기에서 파지 성공 확인됨.
- **한계(이동 물체에서 부각)**:
  1. z를 "첫 mid 확정값"에 상수 고정 → 물체가 z로도 변하면(경사 벨트·비평면·물체 적재 높이 차) 못 따라감.
  2. 첫 mid 확정 **이전**에 근접해 단일 kpt로 시작하면 얼릴 값이 없어 폴백 부재(raw z를 `alpha_z_offset_mode`로 세게 눌러 쓸 뿐).
  3. mid↔단일kpt 전환이 잦으면 z가 오래된 mid 값에 묶여, 그 사이 물체가 실제로 이동했을 때 표면과 어긋날 수 있음.

## 2. 이동 물체 강건성 보완 제안 (구현 아님)

각 항목은 독립 적용 가능하며, **(b)+(c)를 먼저** 넣는 것을 권한다(가장 낮은 리스크로 스파이크 자체를 제거).

| # | 제안 | 구현 개요 | 기대 효과 | 리스크/트레이드오프 |
|---|---|---|---|---|
| a | mid z **저 alpha_z EMA 추종** (상수 고정 완화) | `last_mid_z`를 갱신 시 덮어쓰지 말고 `last_mid_z += a·(z−last_mid_z)`로 아주 느리게만(예 a≈0.05) 따라가게. 여전히 mid 프레임에서만 갱신 | 완만한 z 변화(경사·적재 높이차)를 흡수하면서 평면 가정 유지 | a가 크면 노이즈가 다시 새어듦 → 추종성↔안정성 균형을 실기 튜닝 필요 |
| b | **패치 분산 게이팅** | `patch_median_depth`가 median뿐 아니라 패치 내 유효 depth의 std도 반환 → std가 임계 초과면 그 프레임 z를 `depth_valid=False`로 폐기(median 채택 안 함) | 2026-07-10 CSV의 "같은 픽셀에서 z가 0.01↔0.22m로 튀는" 스파이크를 원천 차단. valid_ratio만으로 못 걸러지던 케이스 커버 | 임계가 낮으면 정상 프레임도 버려 유효율↓ → detect miss 증가 가능. 임계는 거리 의존이라 튜닝 필요 |
| c | **거리 비례 동적 패치** | 9×9 **픽셀** 고정 대신, 목표 실폭(손잡이 ≈1cm)을 픽셀로 환산(`half_px ≈ (target_m/2)·fx/z`)해 패치 크기를 거리로 조절. `reconstruct()`의 half 클램프도 bbox 전체가 아닌 로컬 폭 기준으로 | 원거리에서 패치가 배경/벨트를 물어 median이 쏠리는 것 방지. fx≈210(424×240 추정) 기준 0.25m부터 9px 실폭이 1cm를 넘어 이미 손잡이 폭 초과 | 너무 작은 패치는 depth 홀에 취약(단일 픽셀 문제로 회귀) → 하한(예 3×3) 필요 |
| d | **yaw 축 계산도 같은 게이팅 공유** | `tool_axis_from_depth`는 z-lock과 무관하게 매 프레임 depth를 재샘플하므로 (b)의 분산 게이팅/(c)의 동적 패치를 축 마스크 생성에도 적용 | 근접·가림 구간에서 축(yaw) 노이즈 감소 → 파지 방향 안정 | 마스크 픽셀 수가 줄면 `YAW_MIN_MASK_PX`(50) 미달로 축 계산 포기 빈도↑ |

### 검증 방법(구현 시)
- 회귀: 이동 물체를 원거리→근접으로 통과시키며 `vision.timing_csv_path` CSV를 전/후로 뽑아 `cam_z`(raw)·`base_z`(최종) 프레임간 지터와 스파이크 빈도를 비교(`tools/analyze_timing.py` 재사용).
- 파지 성공률: 동일 벨트 속도에서 N회 반복 파지 성공/허공 파지 카운트.
- 트레이드오프 확인: detect miss rate가 게이팅 강화로 유의하게 오르지 않는지.

### 지금 하지 않는 이유
정지/저속 파지가 이미 `last_mid_z`로 안정적이고, 위 제안들은 이동 물체·비평면 등 아직 요구되지 않은 시나리오의 강건성 개선이다. 실제 이동 물체 파지가 과제로 올라올 때 (b)(c)부터 착수하는 것을 권한다.
